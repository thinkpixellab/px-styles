    @mixin boilerplate($scoped: false, $sanitize: true, $reset: (link,button,lists,headings,paragraph), $fonts: (), $type-styles: null, $code-style: 'code', $link-selector: '.link', $container-selector: '.container', $button-selector: '.button', $hide-selector: '.visually-hidden') { ... }

    Description

    Generates boilerplate css for a site. Generally this only needs to be called once as part of the site's global css. Many components and styles have a soft dependency on this (e.g. may expect that a reset has been applied, etc.). This mixin relies on some shared settings from the site module, so it should be included after that module.

    If true we generate all rules directly within the provides scope rather than use the :root selector.

    		Anyfalse

    		Anytrue

    The list of additional resets to include.

    		Any(link,button,lists,headings,paragraph)
    $fonts

    The list of fonts to be loaded.

    		Any()
    $type-styles

    A map containing a set of type styles where the key is the name and the value is the style. Leave null to use the default config. Set to false to skip.

    		Anynull
    $code-style

    A type-style to apply to code and pre elements (must be contained in the $type-styles map). Set to false to skip.

    		Any'code'
    $link-selector

    A selector to use for generated .link style to. Set to false to skip.

    		Any'.link'
    $container-selector

    toggle-helper

    @mixin toggle-helper($state: null) { ... }

    Description

    Mixin to help create toggle components (checkbox, radiobutton, toggle, togglebutton, etc.) using the common input+label structure. The basic idea behind this pattern is to use an and a 

    <div class="my-checkbox">
    <input type="checkbox" id="cb" />
    <label for="cb">Toggle Me</label>
</div>

    The label and input are bound together by the for property on the label (which refers to the input by id), making the label a proxy for the input. The label can be styled much more readily than the input, so the common practice is to hide the input and style the label but the css rules that allow us to manage states in this situation can be difficult to remember and can be a little complex. This mixin helps to generate those rules.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $state

    The name of the state (e.g. hover, checked, indeterminate, etc.) under which the css in the @content area should apply. The mixin will generate the appropriate rule for that state.

    		Stringnull

    Example

    .my-toggle {

    // do the basic setup (hide inputs, etc.)
    @include toggle-helper(base);

    @include toggle-helper {
        // default state
    }
    @include toggle-helper(checked) {
        // checked state
    }
    @include toggle-helper(hover) {
        // hover state
    }
}

    vue-transition

    @mixin vue-transition($name) { ... }

    Description

    Set of mixins that generate classes for a vue transition. These dot not expose the full power of vue transitions but it does simplify the creation of many by using simplified naming and grouping classes that are commonly defined together. The outer mixin contains the inner (vt-*) mixins.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $name

    The name of the transition

    		String— none

    Example

    // Create a fade transition by definining just the css transition and the hidden state.
 @include vue-transition(fade) {

    // define the transition
    @include vt-transition {
        transition: opacity 500ms;
    }

    // define the hidden state (the non-hidden state is implicty -- the normal state)
    @include vt-hidden {
        opacity: 0;
    }
};
    // Create a slide and fade transition by
 @include vue-transition(slide-x) {

    // define the hidden state (the non-hidden state is implicty -- the normal state)
    @include vt-transition {
        transition: opacity 500ms;
    }

    // the hidden state before entering
    @include vt-hidden-enter {
        transform: translateY(-100px);
        opacity: 0;
    }

    // the hidden after exiting
    @include vt-hidden-leave {
        transform: translateY(100px);
        opacity: 0;
    }
};

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $bases

    The base size or sizes used to generate the typescale

    		Number or List— none
    $ratio

    The ratio used to generate the typescale

    		Number— none

    		Number4

    Map —

    A map of named type sizes, e.g. (small: 0.75rem, -base: 1rem, large: 1.33rem)
